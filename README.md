# Local Risk Pipeline: DuckDB + dbt

This is a project I built to explore how `dbt` and DuckDB handle time-series financial math. It is not meant to be a production-ready pipeline, but rather an example of how to push quantitative logic (like rolling volatility and anomaly detection) directly into the SQL layer.


## The Stack
* **DuckDB:** Local database.
* **dbt (Data Build Tool):** Manages the transformations, window functions, and schema testing.
* **Python:** Pandas and Matplotlib for visualizing the final views.

## How It Works
The pipeline ingests a raw CSV of property sales (artificial data generated by generate_data.py) and applies rolling statistical calculations:

* **Moving Averages & Volatility:** The SQL models use partitioned window functions to calculate a 90-sale rolling average and rolling sample standard deviation for each specific asset class (suburb).
* **Anomaly Detection (Z-Scores):** The pipeline calculates a rolling Z-score to normalize the risk profile.

## Running It Locally

1. **Load the raw data:** `dbt seed` (loads `raw_sales.csv` into DuckDB)
2. **Compile and run the math:** `dbt run` 
3. **Validate the data:** `dbt test` (runs schema assertions, checking for nulls)
4. **Visualize the output:** Run the Python scripts in the `analysis/` folder to query DuckDB and plot the normalized risk profile.
